## Template Usage

1. Copy this template to create a new todo
2. Update the frontmatter:
   - `status`: pending, ready, complete, blocked
   - `priority`: p1 (Critical), p2 (Important), p3 (Nice-to-Have)
   - `issue`: GitHub issue number if applicable
   - `dependencies`: List of related todo files that must complete first
   - `created`: Today's date
   - `updated`: Update this whenever you modify the todo
   - `related_prs`: Pull request numbers if applicable

3. Fill in the sections:
   - **Problem Statement**: What needs to be fixed or implemented
   - **Context**: Background information
   - **Findings**: Investigation results (checkboxes for tracking)
   - **Solutions**: Step-by-step resolution plan
   - **Acceptance Criteria**: Verifiable completion criteria
   - **Work Log**: Chronological record of progress

4. File naming: `[issue-no]-[status]-[priority]-[short-description].md`
   - Example: `123-pending-p2-fix-n-plus-one-queries.md`
   - Example: `no-issue-ready-p1-security-vulnerability.md`
